# 📱 iPhoneから使えるようにする（クラウド公開ガイド）

このアプリを無料のクラウド（**Render**）に置くと、**Macが消えていても、外出先でもiPhoneから**使えるようになります。

> 💡 **保有銘柄の記録はiPhoneの中に保存されます**（クラウドには残りません）。
> なので、無料プランでサーバーが再起動してもあなたの記録は消えず、他人に見られることもありません。

所要時間：だいたい15〜20分。難しいコマンド入力は不要です。

---

## 全体の流れ

```
①GitHubにコードを置く  →  ②Renderで公開  →  ③iPhoneで開く
```

無料サービスを2つ（GitHub と Render）使います。どちらも無料で、クレジットカード登録も不要です。

---

## ① GitHub にコードをアップロード

1. [github.com](https://github.com/) にアクセスし、**Sign up** で無料アカウントを作成（メール・パスワード・ユーザー名）。
2. ログイン後、右上の「**＋**」→「**New repository**」をクリック。
3. 「Repository name」に好きな名前（例：`kabu-watch`）を入力 → 下の「**Create repository**」を押す。
4. 次の画面で「**uploading an existing file**」というリンクをクリック。
5. Mac の `デスクトップ/株分析` フォルダを開き、**中身を全部**（`app` フォルダ・`render.yaml`・`README.md` など）を、ブラウザの点線枠の中へ**ドラッグ＆ドロップ**。
   - フォルダごとドラッグすればOK（中の `app/static/...` も一緒に上がります）。
6. 下の「**Commit changes**」を押す。アップロード完了です。

---

## ② Render で公開する

1. [render.com](https://render.com/) にアクセス →「**Get Started**」。
2. 「**GitHub** で続ける（Sign in with GitHub）」を選び、さっき作ったGitHubでログイン・許可。
3. ダッシュボードで「**New +**」→「**Blueprint**」をクリック。
4. さっきアップロードしたリポジトリ（例：`kabu-watch`）を選んで「**Connect**」。
   - このアプリには `render.yaml` が入っているので、設定は**自動**で読み込まれます。
5. 環境変数 `APP_PIN` の入力欄が出たら、自分だけが知っているPINを入れます。
   - 例：誕生日や `1234` は避け、6文字以上の数字や英字にしてください。
   - このPINを設定すると、公開URLを開く前に認証画面が出ます。
6. 「**Apply**」を押すと、ビルド＆公開が始まります（数分待ちます）。
7. 完了すると、`https://jp-stock-watch-xxxx.onrender.com` のような**URL**が発行されます。これがあなたのアプリのアドレスです。

---

## ③ iPhone で開く・ホーム画面に追加

1. iPhoneの **Safari** で、②で発行された `https://....onrender.com` を開く。
2. 認証画面が出たら、ユーザー名は任意（例：`me`）、パスワードに `APP_PIN` で設定したPINを入力します。
3. 下の「**共有**」ボタン（□に↑）→「**ホーム画面に追加**」を選ぶ。
4. これで、**アプリのアイコン**がホーム画面にでき、ふつうのアプリのように起動できます。

---

## 知っておくと安心なこと（無料プランの特徴）

- **しばらく使わないと“お休み”します**：15分ほどアクセスが無いとサーバーが休止し、次に開くとき**起動に20〜50秒**かかります（その間はローディング表示）。一度開けば快適に動きます。
- **記録は消えません**：保有銘柄・ウォッチリストは**iPhoneの中**に保存されるので、サーバーが再起動しても残ります。
- **公開URLはPINで保護**：`APP_PIN` を設定していれば、URLを知っている人でもPINなしでは開けません。PINは家族以外に共有しないでください。
- **株価・ニュースの取得**：Renderはインターネットへの接続が自由なので、Yahooの株価もGoogleニュースも問題なく取得できます。

---

## うまくいかないときは

| 症状 | 対処 |
|------|------|
| Renderのデプロイが赤く失敗する | ログ画面の最後の数行をコピーして教えてください。だいたい設定の小さなズレです。 |
| 開いても真っ白／読み込み中のまま | 初回は起動に時間がかかります。1分ほど待って再読み込み。 |
| 株価が「取得失敗」 | 少し待って画面の「更新」。一時的なアクセス制限の場合があります。 |

## 補足：自宅Wi-Fi内だけで使う方法（クラウド不要）

外出先で使わず、**Macを起動したまま**にできるなら、クラウドは不要です。
`iPhone起動.command` で立ち上げると、ターミナルに

```
iPhone等で開く: http://192.168.x.x:8765/
```

と表示されるので、**同じWi-Fiのうち**ならそのURLをiPhoneのSafariに入れるだけで使えます（Macがスリープ／電源オフだと使えません）。
